# 🛡️ PROJECT OVERVIEW & GOALS
CyberGuard AI is a multi-vector threat detection and real-time prevention platform designed for enterprise threat monitoring. It ingests URLs, Emails, Network logs, and Web application requests, classifies them using trained machine learning classifiers, enriches them with threat intelligence, and correlates attacks to dynamically adjust risk scores. It includes an active Intrusion Prevention System (IPS) that blocks malicious entities at the middleware layer (403 Forbidden) and runs automated cleanups. It also tracks user activities using User Behavior Analytics (UBA) to detect anomalies (like Impossible Travel or API spikes).

The system is multi-tenant and workspace-isolated. Each workspace has independent API keys, rate limits, monthly scan quotas, and security configuration profiles.

- **PreventionMiddleware**: Intercepts all incoming API and workspace gateway requests. Extracts the client's source IP, queries active blocklists (`BlockedEntity`), and blocks requests immediately with a `403 Forbidden` response if the entity is active in the blocklist.

### 4. AI AGENT ORCHESTRATOR (`SecurityAgent`)
- Acts as the central threat evaluation pipeline (`orchestrator.py`).
- Routes incoming payloads to vector detectors in parallel using `asyncio.gather`.
- Coordinates threat intelligence lookups (`ThreatIntelService`), cross-vector correlation (`CorrelationEngine`), and User Behavior Analytics telemetry.
- Synthesizes vector analysis reports, computes the unified threat risk score (0-100), maps classifications to MITRE ATT&CK vectors, generates developer-friendly text explanations, and issues automated alerts.

### 5. MACHINE LEARNING DETECTORS
(Refer to the detailed "MACHINE LEARNING DETECTORS & FEATURE ENGINEERING" section below)

### 6. THREAT INTELLIGENCE (`ThreatIntelService`)
- Performs real-time lookups on indicators (IPs, domains, email addresses, URL strings) against local and cached threat blacklists with specific TTL configurations.

### 7. CORRELATION ENGINE (`CorrelationEngine`)
- Scans historical records within a rolling 24-hour window to find overlapping indicators (e.g. an IP address flagged in a Network Scan also appearing in a Web payload scan, or a URL scanned in an email matching a known phishing site).
- Applies an escalation boost factor (e.g. +10 default boost, or 1.15x to 1.3x multiplier) to the final risk score when cross-vector indicators match.

### 8. INTENSE ALERT SYSTEM (`AlertService`)
- Automatically escalates threat severity (LOW, MEDIUM, HIGH, CRITICAL) based on the calculated risk score.
- Restricts duplicate alerts for the same indicator using a 60-minute duplicate cooldown.
- Distributes alert notifications via webhooks and SMTP email templates to configured security contacts.

### 9. INTRA-SYSTEM SCHEDULER (`prevention_scheduler.py`)
- Runs a background thread cron process every 5 minutes to release blocked IPs whose block lifetimes (`blocked_until`) have expired, restoring their network access.

### 10. USER BEHAVIOR ANALYTICS (UBA) (`UserBehaviorAnalyticsService`)
- Evaluates active user sessions against baseline profiles, calculating anomaly scores based on deviations from normal activities (impossible travel speeds, brute-force attempts, off-hours access, API spikes).

### 11. EXPLAINABILITY & MITRE ATT&CK (`mitre_mapping_service.py` & `threat_explanation_service.py`)
- Maps classified threat vectors to formal MITRE ATT&CK technique IDs (e.g., T1566 Phishing, T1190 Exploit Public-Facing Application).
- Synthesizes clear, readable textual explanations detailing why an alert was triggered and provides recommended analyst remediation steps.

### 12. EXECUTIVE PDF GENERATOR (`pdf_report_service.py`)
- Compiles weekly and monthly security posture summaries. Generates a formatted PDF report with workspace KPIs, active threat breakdown charts, UBA event histories, and compliance audit summaries.

### A. URL Scanner Model
- **Source Code**: [url_detector.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/detectors/url_detector.py) | [url_feature_extractor.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/detectors/url_feature_extractor.py) | [train_url_model.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/train_url_model.py)
- **Classifier**: Random Forest Classifier (`RandomForestClassifier` with 80 estimators, max depth 12, min samples split 10, min samples leaf 5).
- **Training Dataset**: UCI Phishing Websites Dataset (stored in `.arff` format, containing features representing malicious and benign URLs).
- **Target Label Mapping**:
  - `0`: PHISHING WEBSITE (corresponding to Suspicious/Phishing labels mapped from UCI Result values)
  - `1`: LEGITIMATE WEBSITE
- **Features Trained On (30 features total)**:
  The model is trained on 30 lexical, host-based, and page-based indicators extracted from the URL. The list of features includes:
  1. `having_IP`: Binary (1 if the URL contains a raw IP address pattern, 0 otherwise).
  2. `URL_Length`: Integer (length of the entire URL string).
  3. `Shortining_Service`: Binary (1 if the URL is shortened using a known service like bit.ly or tinyurl, 0 otherwise).
  4. `having_At_Symbol`: Binary (1 if the URL contains an `@` character, 0 otherwise).
  5. `double_slash_redirecting`: Binary (1 if the double slash `//` occurs more than once, 0 otherwise).
  6. `Prefix_Suffix`: Binary (1 if a hyphen `-` is present in the domain name, 0 otherwise).
  7. `having_Sub_Domain`: Integer (number of subdomains present, calculated as count of dots `.` in domain name minus 1).
  8. `SSLfinal_State`: Binary (1 if the URL scheme/protocol is `https`, 0 otherwise).
  9. `HTTPS_token`: Binary (1 if the token `https` is present in the netloc/domain name, 0 otherwise).
  10. `port`: Binary (1 if a non-standard port is explicitly specified, 0 otherwise).
  11. *UCI Dataset Placeholders / Metadata Features* (initialized to 0 or 1 baseline values to align input shape with the UCI 30-feature vector):
      `Domain_registeration_length`, `Favicon`, `Request_URL`, `URL_of_Anchor`, `Links_in_tags`, `SFH`, `Submitting_to_email`, `Abnormal_URL`, `Redirect`, `on_mouseover`, `RightClick`, `popUpWidnow`, `Iframe`, `age_of_domain`, `DNSRecord`, `web_traffic`, `Page_Rank`, `Google_Index` (defaults to 1), `Links_pointing_to_page`, `Statistical_report`.
- **Live Overrides & Heuristics**:
  - **Whitelisting**: If a URL contains safe domains (e.g. `google.com`, `github.com`, `microsoft.com`, `apple.com`), it bypasses the model verdict and is labeled **LEGITIMATE WEBSITE** with a minimum 98% confidence.
  - **Phishing Keyword Boost**: If the model predicts **LEGITIMATE WEBSITE** but with a confidence score under 85%, and the URL contains 2 or more suspicious keywords (e.g. `login`, `verify`, `update`, `bank`, `secure`, `account`, `confirm`), the verdict is overridden to **PHISHING WEBSITE** with a confidence score of 95%.

### B. Email Phishing Model
- **Source Code**: [email_detector.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/detectors/email_detector.py) | [merge_and_train_email_model.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/merge_and_train_email_model.py)
- **Classifier**: Multinomial Naive Bayes (`MultinomialNB`).
- **Training Dataset**: A custom merged email corpus built from multiple public email datasets, deduplicated and sampled down to a balanced 40,000-row subset (to avoid memory crashes during training).
- **Target Label Mapping**:
  - `0`: LEGITIMATE EMAIL (HAM)
  - `1`: PHISHING EMAIL (SPAM)
- **Preprocessing Pipeline**:
  - Text input is created by combining the email's Subject and Body: `subject + " " + body`.
  - The text is converted to lowercase, strips leading/trailing whitespaces, replaces URL links matching `http\S+` with the token `" URL "`, and replaces all non-word characters `\W+` with spaces.
- **Features Trained On (1000 features total)**:
  - Text features are extracted using a **TF-IDF Vectorizer** configured with:
    - `max_features=1000` (retains the top 1000 most predictive vocabulary words).
    - `stop_words='english'` (filters out English stop words like 'the', 'is', 'at').
    - `min_df=3` (ignores terms that appear in fewer than 3 email documents).
  - The model trains on a 1,000-column sparse matrix representing the TF-IDF frequencies of these vocabulary words.

### C. Network Intrusion Detection System (IDS) Model
- **Source Code**: [network_detectors.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/detectors/network_detectors.py) | [train_network_ids.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/train_network_ids.py) | [network_defense.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/src/services/network_defense.py)
- **Classifier**: Random Forest Classifier (`RandomForestClassifier` with 100 estimators, max depth 15).
- **Training Dataset**: Cleaned NSL-KDD Network Intrusion Dataset.
- **Preprocessing Pipeline**:
  - Cleans input data by replacing infinity values (`inf`, `-inf`) with `NaN`, drops rows with missing data, and selects only numerical columns.

- **Features Trained On (76 network flow features)**:
  The model is trained on 76 numeric metrics of network flows. The exact list of features includes:
  `Flow Duration`, `Total Fwd Packet`, `Total Bwd packets`, `Total Length of Fwd Packet`, `Total Length of Bwd Packet`, `Fwd Packet Length Max`, `Fwd Packet Length Min`, `Fwd Packet Length Mean`, `Fwd Packet Length Std`, `Bwd Packet Length Max`, `Bwd Packet Length Min`, `Bwd Packet Length Mean`, `Bwd Packet Length Std`, `Flow Bytes/s`, `Flow Packets/s`, `Flow IAT Mean`, `Flow IAT Std`, `Flow IAT Max`, `Flow IAT Min`, `Fwd IAT Total`, `Fwd IAT Mean`, `Fwd IAT Std`, `Fwd IAT Max`, `Fwd IAT Min`, `Bwd IAT Total`, `Bwd IAT Mean`, `Bwd IAT Std`, `Bwd IAT Max`, `Bwd IAT Min`, `Fwd PSH Flags`, `Bwd PSH Flags`, `Fwd URG Flags`, `Bwd URG Flags`, `Fwd Header Length`, `Bwd Header Length`, `Fwd Packets/s`, `Bwd Packets/s`, `Packet Length Min`, `Packet Length Max`, `Packet Length Mean`, `Packet Length Std`, `Packet Length Variance`, `FIN Flag Count`, `SYN Flag Count`, `RST Flag Count`, `PSH Flag Count`, `ACK Flag Count`, `URG Flag Count`, `CWR Flag Count`, `ECE Flag Count`, `Down/Up Ratio`, `Average Packet Size`, `Fwd Segment Size Avg`, `Bwd Segment Size Avg`, `Fwd Bytes/Bulk Avg`, `Fwd Packet/Bulk Avg`, `Fwd Bulk Rate Avg`, `Bwd Bytes/Bulk Avg`, `Bwd Packet/Bulk Avg`, `Bwd Bulk Rate Avg`, `Subflow Fwd Packets`, `Subflow Fwd Bytes`, `Subflow Bwd Packets`, `Subflow Bwd Bytes`, `FWD Init Win Bytes`, `Bwd Init Win Bytes`, `Fwd Act Data Pkts`, `Fwd Seg Size Min`, `Active Mean`, `Active Std`, `Active Max`, `Active Min`, `Idle Mean`, `Idle Std`, `Idle Max`, `Idle Min`.
- **Heuristic Triggers & Rule-Based Network Defense (`NetworkDefenseService`)**:
  In addition to the ML model output, the system evaluates incoming traffic flow using high-performance security rule heuristics to dynamically compute an anomaly score:
  - **High-Traffic Rate Rule**: Flow Rate > 100,000 Bytes/s OR Packet Rate > 5,000 Packets/s increases the anomaly score by **25 points**.
  - **Brute-Force Rule**: Failed login attempts >= 5 increases the anomaly score by **30 points**.
  - **Suspicious Protocol Rule**: Usage of protocols other than standard TCP, UDP, or ICMP increases the anomaly score by **20 points**.
  - **Syn Flood Rule**: SYN Flag Count >= 20 in the flow increases the anomaly score by **20 points**.
  - **Repeat Offender IP Rule**: If the source IP was logged in the network prevention audit tables at least 3 times in the last 10 minutes, the score increases by **25 points**.
  - **Port Scanning Pattern Rule**: If the source IP attempted connections to 10 or more distinct destination ports in the last 15 minutes, the score increases by **35 points**.
  - *Dynamic Intervention thresholds*:
    - **Score >= 70**: High-risk trigger. Severity is set to **CRITICAL**, and the system automatically enforces temporary blocking and rate limiting.
    - **Score >= 45**: Medium-high-risk trigger. Severity is set to **HIGH**, enforcing active rate limiting on the IP.
    - **Score >= 20**: Low-medium trigger. Severity is set to **MEDIUM**.

### D. Web Attack Scanner
- **Source Code**: [web_detector.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/detectors/web_detector.py) | [train_web_attack_model.py](file:///c:/Users/Farooq/Desktop/Final_Year_Project/train_web_attack_model.py) | [detection_service.py:L55-L101](file:///c:/Users/Farooq/Desktop/Final_Year_Project/src/services/detection_service.py#L55-L101)
- **Classifier**: High-fidelity Regular Expression (Regex) Scanner + Fallback Classifier Model.
- **Target Label Mapping**:
  - `0`: LEGITIMATE REQUEST (SECURE)
  - `1`: PHISHING / MALICIOUS REQUEST
- **Heuristic Rules & RegEx Feature Scopes**:
  Live analysis checks web request payloads against a series of compiled regex engines:
  - **SQL Injection (SQLi) Pattern**: Matches quotes (`'`), comment dashes (`--`), hashes (`#`), hex values (`%27`, `%23`), and SQL command keywords (`union`, `select`, `insert`, `drop`, `update`, `delete`, `truncate`, `sleep`, `benchmark`). Overrides to **SQL INJECTION** (severity: **CRITICAL**, confidence: **99.0%**).
  - **Cross-Site Scripting (XSS) Pattern**: Matches tags (`<script>`), encoded tags (`%3Cscript%3E`), javascript links (`javascript:`), element handlers (`onerror=`, `onload=`), alert triggers (`alert(`), and malicious images (`<img `). Overrides to **CROSS-SITE SCRIPTING (XSS)** (severity: **CRITICAL**, confidence: **99.0%**).
  - **Local File Inclusion (LFI) / Path Traversal Pattern**: Matches directory jumps (`../`, `..\`), encoded jumps (`%2e%2e%2f`, `%2e%2e/`, `../%2f`), and sensitive system files (`/etc/passwd`). Overrides to **PATH TRAVERSAL** (severity: **HIGH**, confidence: **95.0%**).
- **Original Model Training Exception**:
  - *Accidental Training Setup*: The model script `train_web_attack_model.py` originally trained `web_attack_model.pkl` on `Phising_Training_Dataset.csv` (a URL phishing dataset) instead of web payload logs.
  - *Mitigation & Fallback*: To prevent false positives, the system uses the high-precision regex scanners above. When payloads do not match any of these regex threat patterns, the system calls the fallback model `predict_web_attack({})` with empty dictionary features, and overrides the verdict to **LEGITIMATE REQUEST** (severity: **SECURE**, confidence: **90.0%**).

# 🗄️ DATABASE SCHEMA REFERENCE (SQLAlchemy Models)

The database schema manages SaaS state, telemetry, and prevention rules across 11 tables:

1. **Workspace**: Defines tenants. Includes `id` (UUID), `name`, `tier` (free/pro/enterprise), `monthly_quota` (scan limits), `rate_limit_rpm` (RPM configuration), and `created_at`.
2. **User**: Represents platform users. Includes `id` (UUID), `workspace_id` (FK), `email`, `hashed_password`, `full_name`, `role` (admin, developer, viewer), `is_active`, and `created_at`.
3. **APIKey**: Client secrets for accessing APIs programmatically. Includes `id` (UUID), `workspace_id` (FK), `key_hash` (SHA-256), `label`, `is_active`, `created_at`, and `last_used`.
4. **ScanHistory**: Records of all vector scans. Includes `id` (UUID), `workspace_id` (FK), `user_id` (FK), `input_type` (url/email/network/web), `entity` (IP/Domain/Sender), `entities` (JSON list of extracted items), `attack_type`, `severity`, `ml_confidence`, `intelligence_hit` (bool), `correlation_hit` (bool), `prevention_triggered` (bool), `risk_score`, `verdict`, `explanation` (JSON), `mitre_mappings` (JSON), `details` (JSON), and `created_at`.
5. **AuditLog**: Administrative action trail. Includes `id` (UUID), `workspace_id` (FK), `user_id` (FK), `action` (e.g. resolve_alert, unblock_ip), `module`, `status` (success/warning/failure), `event_metadata` (JSON), and `created_at`.
6. **UserBehaviorProfile**: Baseline behavior configuration per user. Includes `id` (UUID), `workspace_id` (FK), `user_id` (FK), `average_daily_logins`, `average_api_calls`, `common_ip_addresses` (JSON), `common_locations` (JSON), `common_login_hours` (JSON), `baseline_risk_score`, `created_at`, and `updated_at`.
7. **UserBehaviorEvent**: Individual user actions tracked by UBA. Includes `id` (UUID), `workspace_id` (FK), `user_id` (FK), `event_type`, `ip_address`, `location`, `endpoint_accessed`, `timestamp`, `anomaly_score`, `risk_level`, and `explanation` (JSON).
8. **ThreatIntel**: Threat indicators blacklist cache. Includes `id` (UUID), `entity_value` (malicious IP/domain/email), `entity_type` (ip/domain/email), `threat_type` (phishing/malware/botnet), `risk_level`, `source` (local/external feed), `last_synced`, and `is_active`.
9. **Alert**: High-priority alerts generated by SecurityAgent. Includes `id` (UUID), `workspace_id` (FK), `user_id` (FK), `scan_history_id` (FK), `alert_type`, `severity`, `title`, `description`, `entity`, `entity_type`, `source_vector` (URL/EMAIL/NETWORK/WEB/UBA), `risk_score`, `ml_confidence`, `indicators` (JSON), `correlated_events` (count), `recommended_action`, `resolved_status`, `resolved_at`, `resolved_by` (FK), `resolution_notes`, `notification_sent`, `email_sent`, `webhook_sent`, `created_at`, and `updated_at`.
10. **AlertHistory**: State change history logs for alerts. Includes `id` (UUID), `alert_id` (FK), `workspace_id` (FK), `user_id` (FK), `action` (escalate/resolve), `previous_severity`, `new_severity`, `notes`, and `created_at`.
11. **BlockedEntity**: Contains lists of blocked IPs, URL paths, and domains enforced by IPS. Includes `id` (UUID), `workspace_id` (FK), `entity`, `entity_type` (ip/url/domain), `severity`, `reason`, `blocked_until` (timestamp), `auto_generated` (bool), `resolved_status` (bool), `prevention_reason`, `related_alert_id` (FK), `related_scan_id` (FK), `blocked_request_count`, `created_at`, `created_by` (FK), `unblocked_at`, and `unblocked_by` (FK).
